# Weekly Briefing Skill

Generate comprehensive CEO Briefing reports with business audit, revenue tracking, and proactive suggestions.

## Usage

```
/weekly-briefing
```

Or specify a period:
```
/weekly-briefing --period "2026-01-01 to 2026-01-07"
```

## What this skill does

1. Audits completed tasks from the week
2. Analyzes business metrics and revenue
3. Identifies bottlenecks and delays
4. Reviews expenses and subscriptions
5. Generates proactive suggestions
6. Creates CEO Briefing document in Briefings/ folder
7. Tracks upcoming deadlines and goals

## Data Sources

### Tasks Analysis
- **Source**: Done/ folder
- **Extract**: Completed tasks, timestamps, completion time
- **Analyze**: Velocity, bottlenecks, patterns

### Revenue Tracking
- **Source**: Accounting/ folder, invoices sent
- **Extract**: Payment amounts, dates, clients
- **Analyze**: Revenue vs. goals, trends

### Expense Analysis
- **Source**: Bank transactions, expense logs
- **Extract**: Subscriptions, recurring costs
- **Analyze**: Cost optimization opportunities

### Goal Progress
- **Source**: Business_Goals.md
- **Extract**: Targets, milestones
- **Analyze**: Progress toward objectives

## Briefing Generation Process

### Step 1: Gather Completed Tasks
- Scan Done/ folder for the period
- Extract task completion dates
- Categorize by type (client, internal, admin)
- Calculate completion metrics

### Step 2: Analyze Revenue
- Sum invoices paid in period
- Compare to Business_Goals.md targets
- Calculate MTD (month-to-date) progress
- Identify revenue trends

### Step 3: Identify Bottlenecks
- Compare expected vs. actual completion times
- Flag tasks that took >2x expected time
- Note any blocked or delayed items
- Identify patterns in delays

### Step 4: Review Expenses
- Analyze bank transactions for period
- Identify subscription costs
- Flag unused or underused services
- Calculate burn rate

### Step 5: Generate Suggestions
Based on analysis, provide:
- **Cost Optimization**: Cancel unused subscriptions
- **Revenue**: Follow up on pending invoices
- **Process**: Automate repetitive tasks
- **Growth**: Invest in high-ROI activities

### Step 6: List Upcoming Deadlines
- Extract from Business_Goals.md
- Check Plans/ for active projects
- Note any at-risk deadlines
- Prioritize by importance

### Step 7: Calculate Key Metrics
Track these metrics:
- Client response time
- Invoice payment rate
- Software costs
- Task completion rate
- Revenue vs. target

### Step 8: Create Briefing Document
- Write to Briefings/YYYY-MM-DD_Monday_Briefing.md
- Use clear, executive-friendly language
- Include actionable items
- Add checkboxes for decisions needed

## Scheduling

### Recommended Schedule
- **Weekly**: Monday 7:00 AM (before work week)
- **Monthly**: First Monday of month (comprehensive)
- **Quarterly**: First week of quarter (strategic)

### Automation Options

**Windows Task Scheduler:**
```xml
<Task>
  <Trigger>
    <CalendarTrigger>
      <StartBoundary>2026-01-06T07:00:00</StartBoundary>
      <ScheduleByWeek>
        <DaysOfWeek>
          <Monday />
        </DaysOfWeek>
        <WeeksInterval>1</WeeksInterval>
      </ScheduleByWeek>
    </CalendarTrigger>
  </Trigger>
  <Actions>
    <Exec>
      <Command>claude</Command>
      <Arguments>/weekly-briefing</Arguments>
    </Exec>
  </Actions>
</Task>
```

**macOS/Linux cron:**
```bash
# Weekly briefing every Monday at 7 AM
0 7 * * 1 claude /weekly-briefing
```

## Subscription Audit Rules

Flag for review if:
- No login/activity in 30 days
- Cost increased >20% without approval
- Duplicate functionality with another tool
- Below 50% utilization (if trackable)

## Security Notes

- Briefing contains sensitive financial data
- Store in secure Briefings/ folder
- Never share externally without review
- Keep 90-day rolling history
